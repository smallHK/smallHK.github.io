# java HashMap put操作

HashMap使用putVal方法实现Map接口的put方法。

public V put(K key, V value) {
    return putVal(hash(key), key, value, false, true);
}

# putVal

判断table是否为null，或长度是否为0，如果满足两者之一，resize哈希表，获取新长度。

if ((tab = table) == null || (n = tab.length) == 0)
    n = (tab = resize()).length;

hash与表长度执行与操作，然后获取对应的元素，如果不存在对应的元素，直接将键值结点放入表内。

if ((p = tab[i = (n - 1) & hash]) == null)
    tab[i] = newNode(hash, key, value, null);

如果发生冲突，首先判断冲突的结点与传入的hash是否相同且键是否相同，如果满足两者条件，直接获取该元素。

Node<K,V> e; K k;
if (p.hash == hash &&
    ((k = p.key) == key || (key != null && key.equals(k))))
    e = p;

如果找到的结点并非是指定的key，即无法直接更新，判断是否为树节点，如果是树节点，执行设置树结点。

else if (p instanceof TreeNode)
    e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);

如果目标结点为链表结点，遍历链表，判断是否到达链表尾，如果到达表尾部，尾部设置为新节点。如果节点数超过树化阈值（默认8），执行树化，跳出迭代。

for (int binCount = 0; ; ++binCount) {
    if ((e = p.next) == null) {
        p.next = newNode(hash, key, value, null);
        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
            treeifyBin(tab, hash);
        break;
    }
    //...
}

如果当前结点为目标结点，直接跳出迭代。

if (e.hash == hash &&
    ((k = e.key) == key || (key != null && key.equals(k))))
    break;
p = e;

判断是否执行更新操作，如果可执行更新，且旧值为null，执行更新。执行回调，返回旧值。

if (e != null) { // existing mapping for key
    V oldValue = e.value;
    if (!onlyIfAbsent || oldValue == null)
        e.value = value;
    afterNodeAccess(e);
    return oldValue;
}

修改计数加一。

++modCount;

尺寸加一，如果尺寸大于阈值，执行resize。
