# HashTable源码解析——put操作

HashTable通过put操作存储键值映射。put操作本身加synchronized。

首先判断是否为null，如果为null，抛出null异常。

if (value == null) {
    throw new NullPointerException();
}

获取key的哈希值，然后通过hash计算索引

int hash = key.hashCode();
int index = (hash & 0x7FFFFFFF) % tab.length;

获取从内部维护的table获取索引处的节点。由于Entry为列表，逐个遍历链表的结点，直到获取为null的结点。迭代中，如果该点的hash与实体的hash相同，并且两者间相同，则将旧的值返回，新的值赋予原Entry。

Entry<K,V> entry = (Entry<K,V>)tab[index];
for(; entry != null ; entry = entry.next) {
    if ((entry.hash == hash) && entry.key.equals(key)) {
        V old = entry.value;
        entry.value = value;
        return old;
    }
}

如果完成遍历，也没有发现旧结点，则添加新结点，然后返回null。

addEntry(hash, key, value, index);

# HashTable添加Entry

添加结点时，首先获取表。

如果表内结点的总数超过了阈值，则引发刷新操作。刷新之后，获取刷新后的表、哈希值，并且计算新的索引。

if (count >= threshold) {
    // Rehash the table if the threshold is exceeded
    rehash();

    tab = table;
    hash = key.hashCode();
    index = (hash & 0x7FFFFFFF) % tab.length;
}
