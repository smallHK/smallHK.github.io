# ThreadPoolExecutor源码解析——关闭线程池

关闭线程池首先会通过安全管理器检查关闭访问的权限。

SecurityManager security = System.getSecurityManager();
if (security != null) {
    security.checkPermission(shutdownPerm);
    for (Worker w : workers)
        security.checkAccess(w.thread);
}

将线程池的状态转为关闭状态。

advanceRunState(SHUTDOWN);

中断所有的空闲工作者。该操作会再次进入锁，通过遍历workers集，获取所有的工作者，并且判断每个工作者的中断状态。如果工作者处于非中断状态，并且成功锁定，则中断该工作者的线程。

执行关闭钩子，并且尝试终止线程。

# 尝试终止线程池

首先获取线程池的状态。

如果线程池满足以下三种条件，则不会执行终止

  • 线程池处于运行状态
  • 线程池已经处于终止或整理状态
  • 线程池并没有处于STOP状态，并且工作队列不为空
int c = ctl.get();
if (isRunning(c) ||
    runStateAtLeast(c, TIDYING) ||
    (runStateLessThan(c, STOP) && ! workQueue.isEmpty()))
    return;

如果工作者数量不为0。中断一个空闲工作者。

if (workerCountOf(c) != 0) { // Eligible to terminate
    interruptIdleWorkers(ONLY_ONE);
    return;
}

实际的终止操作，即将线程池状态转为整理状态，执行终止钩子。随后将线程池转为终止状态，唤醒所有的终止条件。

if (ctl.compareAndSet(c, ctlOf(TIDYING, 0))) {
    try {
        terminated();
    } finally {
        ctl.set(ctlOf(TERMINATED, 0));
        termination.signalAll();
    }
    return;
}
