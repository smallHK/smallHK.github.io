# ThreadPoolExecutor源码解析——Worker执行任务

worker执行任务任务，由内置的thread线程start触发。

public void run() {
    runWorker(this);
}

Worker本身实现Runnable接口，该接口的实现方法，为调用线程池的runWorker方法，该方法负责执行任务。

runWorker的主体为一个迭代，迭代条件为该工作者创建时赋予的首个firstTask以及阻塞队列中存放的任务。

while (task != null || (task = getTask()) != null) {
    ...
}

当成功获取任务之后，判断线程池状态是否为STOP，如果线程池已经STOP，则中断该线程。

if ((runStateAtLeast(ctl.get(), STOP) ||
        (Thread.interrupted() &&
        runStateAtLeast(ctl.get(), STOP))) &&
    !wt.isInterrupted())
    wt.interrupt();

执行任务，并且在任务执行前后运行钩子beforeExecute、afterExecute。为了确保线程不被异常中断，使用Throwable将所有可能的异常以及错误全部捕获，异常可以通过afterExecute获取。最后完成执行时，增加工作者的完成任务数。

try {
    beforeExecute(wt, task);
    try {
        task.run();
        afterExecute(task, null);
    } catch (Throwable ex) {
        afterExecute(task, ex);
        throw ex;
    }
} finally {
    task = null;
    w.completedTasks++;
    w.unlock();
}

最后，在工作者退出获取任务迭代时，将会执行工作者退出方法processWorkerExit。注意，runWoker时，会维护一个标识，表示工作者是否正常退出。当工作者正常退出获取任务迭代时，将会将此标记设置为false，表示正常退出，该标记会被传入退出方法。

如果是正常退出，则不会减少工作者数量。

if (completedAbruptly) // If abrupt, then workerCount wasn't adjusted
    decrementWorkerCount();

增加线程池自身完成的工作计数，然后将工作者从工作者集合中移除。由于该过程涉及对共享变量的处理，对该过程执行锁同步处理。

final ReentrantLock mainLock = this.mainLock;
mainLock.lock();
try {
    completedTaskCount += w.completedTasks;
    workers.remove(w);
} finally {
    mainLock.unlock();
}

尝试终止线程池，即确认线程池是否处于运行状态，如果不是，则不进行终止，如果并非TIDYING或RUNNABLE状态，并且工作队列为空，则尝试进行终止。

if (isRunning(c) ||
    runStateAtLeast(c, TIDYING) ||
    (runStateLessThan(c, STOP) && ! workQueue.isEmpty()))
    return;

如果工作者总数不为0，则终止一个工作者。

if (workerCountOf(c) != 0) { // Eligible to terminate
    interruptIdleWorkers(ONLY_ONE);
    return;
}

如果工作者数量为0，则尝试终止线程池。即将线程池设置为TIDYING状态，工作者设置为0，执行终止钩子terminated。钩子执行完成后，将池状态设置为TERMINATED。并且通知所有陷入termination条件等待的线程。

if (ctl.compareAndSet(c, ctlOf(TIDYING, 0))) {
    try {
        terminated();
    } finally {
        ctl.set(ctlOf(TERMINATED, 0));
        termination.signalAll();
    }
    return;
}

尝试关闭后，线程池获取状态，如果池并没有关闭，判断是否处于正常关闭状态。如果是正常关闭退出，并且工作者数量大于最小值，则不考虑增加新的非核心工作者，否则考虑增添非核心工作者。

int c = ctl.get();
if (runStateLessThan(c, STOP)) {
    if (!completedAbruptly) {
        int min = allowCoreThreadTimeOut ? 0 : corePoolSize;
        if (min == 0 && ! workQueue.isEmpty())
            min = 1;
        if (workerCountOf(c) >= min)
            return; // replacement not needed
    }
    addWorker(null, false);
}
