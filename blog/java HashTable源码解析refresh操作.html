# HashTable源码解析——refresh操作、get操作

# refresh操作

首先获取旧表的容积，即数组长度，以及旧表本身。

int oldCapacity = table.length;
Entry<?,?>[] oldMap = table;

将原容积翻倍加1，用作新容积。当新容器超过最大数组尺寸时，并且旧容积等于最大数组尺寸时，直接返回，否则，新的容积为最大数组尺寸。

int newCapacity = (oldCapacity << 1) + 1;
if (newCapacity - MAX_ARRAY_SIZE > 0) {
    if (oldCapacity == MAX_ARRAY_SIZE)
        // Keep running with MAX_ARRAY_SIZE buckets
        return;
    newCapacity = MAX_ARRAY_SIZE;
}

初始化一个新的Entry数组用作新的map。增加修改计数modCount。新的阈值被设置为新容积与负载因子的乘积，以及最大数组尺寸+1之间的最小值。哈希表设置为新表。

Entry<?,?>[] newMap = new Entry<?,?>[newCapacity];

modCount++;
threshold = (int)Math.min(newCapacity * loadFactor, MAX_ARRAY_SIZE + 1);
table = newMap;

迭代旧表的每一个数组元素。随后遍历每个元素位置上的链表，将链表上的每个元素重新计算哈希值以及索引，将其作为链表首部元素插入新表中。

# get操作

get操作与put操作相同，使用synchronized关键字加锁。

首先获取table，随后获取键的hash值，并用hash计算出对应的表索引。

Entry<?,?> tab[] = table;
int hash = key.hashCode();
int index = (hash & 0x7FFFFFFF) % tab.length;

遍历Entry链表上的所有节点，逐个使用equals比较键值，返回命中元素值。

for (Entry<?,?> e = tab[index] ; e != null ; e = e.next) {
    if ((e.hash == hash) && e.key.equals(key)) {
        return (V)e.value;
    }
}

完成迭代后未找到值，则返回null。
