# Spring IoC容器 源码解析——注解bean定义读取器注册bean

注解配置ApplicationContext时，需要使用AnnotatedBeanDefinitionReader注册注解bean类。该过程会调用读取器的register方法。

首先明确读取器的实例化方法。该类没有实现任何接口，也没有继承任何类，通过构造构建实例。

public class AnnotatedBeanDefinitionReader

该类初始化过程

public AnnotatedBeanDefinitionReader(BeanDefinitionRegistry registry, Environment environment) {
    Assert.notNull(registry, "BeanDefinitionRegistry must not be null");
    Assert.notNull(environment, "Environment must not be null");
    this.registry = registry;
    this.conditionEvaluator = new ConditionEvaluator(registry, environment, null);
    AnnotationConfigUtils.registerAnnotationConfigProcessors(this.registry);
}

该类会使用传入了bean注册表实例化自身registry字段，此处一般为实现了BeanDefinitionRegitry的ApplicationContext。之后通过注册表以及注册表的环境构建条件判断器实例化conditionEvaluator字段。最将注册表注册注解配置处理器。

注册过程会以传入的component类作为参数逐个调用registryBean方法。

public void register(Class<?>... componentClasses) {
    for (Class<?> componentClass : componentClasses) {
        registerBean(componentClass);
    }
}

# doRegiterBean

注册过程首先使用bean类实例化一个注解通用Bean定义。使用条件判断器根据该bean定义的注解信息判断是否应该跳过该bean，如果跳过直接返回。

AnnotatedGenericBeanDefinition abd = new AnnotatedGenericBeanDefinition(beanClass);
if (this.conditionEvaluator.shouldSkip(abd.getMetadata())) {
    return;
}

为bean定义设置实例提供器。随后通过scope元数据解析器分析bean定义的scope元数据设置bean的scope。使用读取器的bean名称生成器使用bean定义以及注册表，生成bean的名称。

abd.setInstanceSupplier(supplier);
ScopeMetadata scopeMetadata = this.scopeMetadataResolver.resolveScopeMetadata(abd);
abd.setScope(scopeMetadata.getScopeName());
String beanName = (name != null ? name : this.beanNameGenerator.generateBeanName(abd, this.registry));

使用注解配置工具，处理bean定义的通用定义注解。

AnnotationConfigUtils.processCommonDefinitionAnnotations(abd);

使用bean的名称以及bean定义构建bean定义holder。通过注解配置工具应用scope代理模式处理定义Holder。

通过Bean定义读取器工具将定义holder注册到注册表中。
