# java CompletableFuture异步执行Supplier

CompletableFuture<String> f = CompletableFuture.supplyAsync(() -> {
    return "Hello World!";
}, service);

CompletableFuture类用于实现异步操作。supplyAsync用于将执行结果，由thenAccept系列方法异步使用。

supplyAsync使用包私有方法asyncSupplyStage实现。

return asyncSupplyStage(screenExecutor(executor), supplier);

执行器使用screenExecutor进行封装。如果通用池无法使用，且执行器为通用池，则使用异步池ThreadPerTaskExecutor。

if (!USE_COMMON_POOL && e == ForkJoinPool.commonPool())
    return ASYNC_POOL;

异步池ThreadPerTaskExecutor使用新线程执行方法。

static final class ThreadPerTaskExecutor implements Executor {
    public void execute(Runnable r) { new Thread(r).start(); }
}

如果screen使用的执行器为null，抛出异常，否则使用传入的执行器。

if (e == null) throw new NullPointerException();
return e;

asyncSupplyStage首先判断传入的Supplier是否为空，如果为空，抛出异常。

if (f == null) throw new NullPointerException();

构建CompletableFuture对象，使用执行器执行AsyncSupply对象，然后返回构建的CompletableFuture对象。

CompletableFuture<U> d = new CompletableFuture<U>();
e.execute(new AsyncSupply<U>(d, f));
return d;

# AsyncSupply

AsynSupply实现了Runnable接口，以及异步完成任务接口。

运行时首先判断两者是否为null，并且将内部维护的字段设置为null。

CompletableFuture<T> d; Supplier<? extends T> f;
if ((d = dep) != null && (f = fn) != null) {
    dep = null; fn = null;
    //...
}

首先判断结果是否为null，如果为null，执行completeValue，如果抛出异常，执行completeThrowable。最后执行postComplete。

if (d.result == null) {
    try {
        d.completeValue(f.get());
    } catch (Throwable ex) {
        d.completeThrowable(ex);
    }
}
d.postComplete();

completeValue使用RESULT的CAS实现。

final boolean completeValue(T t) {
    return RESULT.compareAndSet(this, null, (t == null) ? NIL : t);
}

completeThrowable使用RESULT的CAS实现。

final boolean completeThrowable(Throwable x) {
    return RESULT.compareAndSet(this, null, encodeThrowable(x));
}

# postComplete

循环当前对象的stack非null，并且Future不为当前对象且当前对象的stack不为null。

while ((h = f.stack) != null ||
        (f != this && (h = (f = this).stack) != null)) {
    //...
}

STACK执行CAS。

if (STACK.compareAndSet(f, h, t = h.next)) {
    //...
}

如果维护的Completion不为null，且维护的CompletableFuture也不为当前对象，则当前状态入栈，重新迭代。

if (f != this) {
    pushStack(h);
    continue;
}

如果当前Future是当前对象，则NEXT执行CAS。

NEXT.compareAndSet(h, t, null); // try to detach

执行触发赋予迭代内Future，如果迭代内Future为null，赋予迭代外Future。
