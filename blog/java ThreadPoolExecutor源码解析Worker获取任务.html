# ThreadPoolExecutor源码解析——Worker获取任务

Worker在运行任务时，存在一个获取任务迭代，即根据初始化任务以及getTask获取任务。

获取任务时，首先判断池状态。如果池关闭，并且工作队列为空，则减少工作者数量。

if (runStateAtLeast(c, SHUTDOWN)
    && (runStateAtLeast(c, STOP) || workQueue.isEmpty())) {
    decrementWorkerCount();
    return null;
}

当设置允许核心线程超时，或者工作者数量大于核心工作者尺寸，则设置超时标记timed。如果可超时且已超时，并且工作者数量大于最大池数量，并且工作者数量大于1或者工作队列为空时，减少工作者数量。此过程不断重复。

// Are workers subject to culling?
boolean timed = allowCoreThreadTimeOut || wc > corePoolSize;

if ((wc > maximumPoolSize || (timed && timedOut))
    && (wc > 1 || workQueue.isEmpty())) {
    if (compareAndDecrementWorkerCount(c))
        return null;
    continue;
}

根据是否设置timed，选择是带超时限制地获取任务，还是无限期等待获取任务。当为超时时，获取成功，否则超时。期间如果遭遇中断timeOuted为false。

try {
    Runnable r = timed ?
        workQueue.poll(keepAliveTime, TimeUnit.NANOSECONDS) :
        workQueue.take();
    if (r != null)
        return r;
    timedOut = true;
} catch (InterruptedException retry) {
    timedOut = false;
}
